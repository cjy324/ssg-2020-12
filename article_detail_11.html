제목 : 1강. HTML & CSS & JS의 개념 및 역할
하나의 웹페이지는 **HTML,** **CSS,** **JS**로 구성되어 있다. **HTML,** **CSS,** **JS**는 각각 자신들만의 역할을 담당하고 있는데, 그 역할들을 잘 조합하면 하나의 웹페이지를 만들어 낼 수 있다. 이러한 **HTML,** **CSS,** **JS**을 다루는 것을 **'웹코딩'** 이라 한다. 지금부터 각각의 기본적인 개념과 역할들을 알아보자.
# < HTML > #### 주요 역할 : 문서 내용 담당[배우]
- 웹페이지를 만들기 위한 언어로 웹브라우저 위에서 동작하는 언어다. - html안에는 이미 **기본 태그**와 **속성**들이 저장되어 있다. - CSS는 이 속성들을 창조하는 것이 아닌 변화시켜주는 리모컨 역할을 한다. ``` -HTML의 의미- HT - HyperText, 문서와 문서가 링크로 연결되어 있다. M - Markup, 태그로 이루어져 있다. L - Language ``` # < CSS > #### 주요 역할 : 문서 꾸밈 담당[스타일리스트]
- CSS는 HTML을 아름답게 꾸며주는 **디자이너의 언어**다. - 잘 설계된 CSS는 **재활용이 가능**하며 **테마, 템플릿**의 형태로 **확장**할 수 있다. - 또한 자바스크립트와 연계해 콘텐츠의 내용이나 디자인을 **동적**으로 처리할 경우에도 유용하게 사용된다. ``` -CSS의 의미- C - Cascading, '폭포처럼 떨어져 내리는'이라는 사전적 의미로 상위 요소에서 정의한 속성이 하위 요소로 전달된다는 의미에서 유래했다. SS - StyleSheet, 스타일을 다루는 정체성을 보여준다. ``` # < JS; JavaScript > #### 주요 역할: 감독[감독]
- JavaScript는 **객체(object)** 기반의 스크립트 언어다. - **HTML로는 웹의 내용을 작성**하고, **CSS로는 웹을 디자인**하고, **JavaScript로는 웹의 동작을 구현**할 수 있다. - JavaScript는 **동적**이며, 타입을 명시할 필요가 없는 **인터프리터 언어**다. - 따라서, 컴파일 작업이 필요하지 않고, 소스코드를 바로 실행한다. ``` -Java와 JavaScript- Java와 JavaScript는 그 이름만 놓고 보면 서로 관련이 있는 언어로 생각되기 쉽다. 하지만 두 언어는 서로 직접적인 관련은 없으며, 비슷한 점보다는 다른 점이 훨씬 많다. 문법상 비슷한 부분은 두 언어의 문법이 모두 C언어를 기반으로 만들어졌기 때문이다. (출처: http://www.tcpschool.com/javascript/js_intro_basic) ```
